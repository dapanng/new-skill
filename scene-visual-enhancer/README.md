# 场景视觉提示词优化 Skill — 快速入门

对场景原提示词做光影、摄像机、氛围的正向视觉增强，完全不改变场景内部布局，室内外场景分治，共 8 节规则 + 工作流 + 固定模板。

---

## 内含内容

- **`SKILL.md`** — 技能入口与索引导航（首先读取）。
- **8 个主题 `.md` 文件** — 每节规则独立成文件，按编号 1-8 对应：核心原则、布局保持、正面视觉表达、光影优化、摄像机优化、氛围与色调、室内策略、室外策略。
- **`workflow.md`** — 使用工作流：何时调用、如何套用优化模板、如何校验、输出规范。
- **`templates/enhancement.template.md`** — 固定优化输出模板（每场景统一格式）。

---

## 如何开始

### 步骤 1. 加载 Skill

> 「加载 scene-visual-enhancer 技能。」

会读取 `SKILL.md` 入口，按「加载时必读」顺序加载各主题文件。

### 步骤 2. 提供原提示词

以下情况之一：

**A. 用户已有场景原提示词（含人物/物件/空间/动作/时间天气要素）。**
发送原提示词 — 按第二节锁定布局锚点后开始增强。

**B. 用户只有场景意图。**
用一段话描述，先按第二节提取布局要素（未指定项标注「未指定」），再开始增强。

**C. 用户要求审查既有增强结果。**
发送既有增强输出 — 按 `workflow.md` 的「如何校验」逐项检查 7 个维度。

### 步骤 3. 按工作流套用模板

标准循环（详见 `workflow.md` 的「如何套用优化模板」）：

1. 第一步：锁定原提示词（参考第二节，五类布局要素）。
2. 第二步：判定室内/室外并选择对应策略（参考第七/第八节）。
3. 第三步：光影增强（参考第四节，用第三节的句式模板）。
4. 第四步：机位与氛围增强（参考第五、六节，冲突即放弃）。
5. 第五步：填写 `templates/enhancement.template.md` 并完成布局保真校验。

### 步骤 4. 校验

按 `workflow.md` 的「如何校验」清单逐项检查 7 个维度（布局保真、正向比例、室内外匹配、光影服务情绪、机位不矛盾、氛围不抢焦、模板完整）。

## 对技能的典型请求

| 请求 | 执行行为 |
|---|---|
| 「优化这段场景提示词的光影」 | 读取 `workflow.md` → 按五步流程套用模板，重点参考第四节 |
| 「在不改布局前提下增强画面」 | 按第二节锁定布局锚点 → 第三至六节正向增强 → 末尾布局保真声明 |
| 「这是室内/室外场景，给我视觉增强策略」 | 参考第七节（室内）或第八节（室外）的对应子类型策略 |
| 「把原提示词转为正向视觉增强」 | 参考第三节正面视觉表达规则 + 句式模板 |
| 「审查我的增强结果是否符合规则」 | 按 `workflow.md` 校验清单逐项检查 7 个维度 |
| 「为 HyperFrames / Remotion 视频生成提示词」 | 按工作流产出正向增强提示词，填入「下游使用建议」字段 |

---

## 输出规范

详见 `workflow.md` 的「输出规范」节。核心要求：

- 所有优化输出必须使用 `templates/enhancement.template.md` 的固定模板。
- 原提示词逐字保留。
- 正向增强区不得出现否定句承担增强。
- 末尾必须含「布局保真声明」。
