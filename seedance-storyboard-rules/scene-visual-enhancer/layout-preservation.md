## 二、场景布局保持规则

布局保真是本 Skill 的第一约束。任何光影/机位/氛围增强都不得与原提示词的布局冲突。

### 布局要素清单

| 要素类别 | 提取要点 |
|----------|----------|
| 人物 | 身份、数量、站位（左/中/右/前/后）、朝向、相互距离关系 |
| 物件 | 种类、位置、与人物的距离/朝向关系、是否参与动作 |
| 空间结构 | 墙体、门窗、家具、地形、植被等空间拓扑与方位 |
| 动作意图 | 人物正在执行的动作、动作方向、动作对象 |
| 时间天气 | 原提示词是否已指定时间（清晨/正午/黄昏/夜晚）与天气（晴/阴/雨/雪/雾） |

### 锁定流程

1. 逐项提取：从原提示词中按上表五类要素逐项提取，原文未提及的标注「未指定」。
2. 写入锚点区：将提取结果写入模板的「锁定锚点区」字段。
3. 增强时对照：后续每条光影/机位/氛围增强描述都必须与锚点区对照，不得冲突。
4. 冲突即放弃：若某条增强与锚点冲突，放弃该增强，不得改写原布局迁就增强。

### 禁止项

- 不得新增、删除、移动原提示词中已出现的人物。
- 不得新增、删除、移动原提示词中已出现的关键物件。
- 不得改变空间拓扑（墙体/门窗/家具/地形/植被的相对方位）。
- 不得改变人物的动作意图与朝向。
- 不得改变原提示词已明确的时间天气；若原提示词未指定，可补充，但必须在「布局保真声明」中标注「原提示词未指定，已补充为 X」。

### 冲突判定

判定原则：增强描述若在画面中会导致人物/物件/空间关系与锚点矛盾，即判为冲突。

示例：原提示词「A 站在左、B 站在右」+ 增强「镜头从右侧拍摄 A 的正面」→ 若该机位会让 A 在画面中出现在右侧，则与锚点冲突，应改用「镜头从左侧拍摄 A 的正面」或「镜头正面拍摄 A 与 B 的关系」。
